# 项目架构草案

## 目录结构

```text
apps/
  client/   React + Vite 前端原型与后续正式客户端
  server/   Node.js + WebSocket 服务端
packages/
  shared/   地图、数值、规则、协议和通用类型
docs/       设计文档、示意图和架构说明
```

## 设计原则

### 1. 共享规则单一来源

地图、坦克、武器、回合规则、地形破坏阈值都放在 `packages/shared`。

作用：

- 客户端和服务端共用同一份基础配置
- 数值调整时不需要在多个工程中重复修改
- 便于后续做地图编辑器或配置校验

### 2. 服务端权威判定

最终联机版本中，以下内容由服务端确定：

- 房间成员
- 当前回合
- 坦克位置与状态
- 武器发射结果
- 地形破坏结果
- 胜负和结算

客户端负责：

- 输入采集
- 画面渲染
- 动画补间
- HUD 与交互反馈

### 3. 数据驱动扩展

后续新增内容时，优先新增配置，而不是改死逻辑：

- 新坦克：增加 `TankDefinition`
- 新武器：增加 `WeaponDefinition`
- 新地图：增加 `MapDefinition`
- 新回合模式：扩展 `PrototypeGameConfig`

### 4. 地图兼容性

地图定义至少要支持：

- 世界尺寸
- 背景与主题
- 平台集合
- 底层承接道路
- 出生点
- 相机参数

后续可继续扩展：

- 风场分区
- 特殊平台材质
- 可交互机关
- 地图主题资源

## 近期实现顺序

### 阶段 1：前端可视化原型

- 将当前静态示意换成 Canvas 渲染
- 实现拖动、缩放、镜头跟随和小地图联动

### 阶段 2：核心战斗循环

- 坦克移动、跳跃、落地
- 蓄力与角度控制
- 炮弹飞行与命中

### 阶段 3：地形系统

- 像素层破坏
- 逻辑网格强度统计
- 脆弱平台坍塌

### 阶段 4：联机

- 房间创建与加入
- 回合轮换
- WebSocket 同步
- 服务端权威状态广播
